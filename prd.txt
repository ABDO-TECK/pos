# 🧾 Product Requirements Document (PRD)

## 🛒 Supermarket POS System (PHP Native + React)

---

# 1. 📌 Overview

## 🎯 الهدف

تطوير نظام كاشير (POS) متكامل لإدارة سوبر ماركت يشمل:

* عمليات البيع (Checkout)
* إدارة المنتجات
* إدارة المخزون
* إدارة الموردين
* التقارير والتحليلات
* إدارة المستخدمين

## 👥 المستخدمين المستهدفين

* 🧑‍💼 الكاشير
* 🧑‍💻 مدير النظام
* 🧑‍💼 صاحب السوبر ماركت

---

# 2. 🧱 System Architecture

## 🔹 Frontend

* React (Vite)
* إدارة الحالة: Zustand
* الاتصال: Axios / Fetch

## 🔹 Backend

* PHP Native (Custom MVC Architecture)

## 🔹 Database

* MySQL

## 🔹 Communication

* REST API (JSON)

---

# 3. 🏗️ Backend Architecture (PHP Native)

## 📁 Structure

```bash
/backend
 ├── config/
 ├── core/
 │    ├── Router.php
 │    ├── Database.php
 │    └── Controller.php
 ├── controllers/
 ├── models/
 ├── middleware/
 ├── routes/
 └── helpers/
```

---

## ⚙️ Core Systems

### 🔹 Router System

* استقبال جميع الطلبات عبر index.php
* تحديد route
* توجيه الطلب للـ Controller المناسب

### 🔹 Controller Layer

* استقبال request
* تنفيذ logic
* إرجاع JSON response

### 🔹 Model Layer

* التعامل مع قاعدة البيانات باستخدام PDO
* تنفيذ Queries بشكل آمن

---

# 4. 🔌 API Design

## 📌 Endpoints

### المنتجات:

* GET /api/products
* POST /api/products
* PUT /api/products/{id}
* DELETE /api/products/{id}

### المبيعات:

* POST /api/sales

### التقارير:

* GET /api/reports/daily
* GET /api/reports/monthly

### المستخدمين:

* POST /api/login
* GET /api/user

---

## 📦 Response Format

```json
{
  "status": "success",
  "data": {}
}
```

أو

```json
{
  "status": "error",
  "message": "Error message"
}
```

# 5. 🛒 Core Modules

---

## 5.1 🛒 POS / Checkout System

### 📌 الوصف:

الشاشة الأساسية للبيع — يجب أن تكون ultra fast.

### ⚙️ الوظائف:

#### 🔹 Barcode Scanning

* إدخال barcode في input
* React يبحث محليًا أو عبر API
* إضافة المنتج فورًا

#### 🔹 Cart System

* قائمة المنتجات المختارة
* تعديل الكمية (+ / -)
* حذف منتج

#### 🔹 Pricing Engine

* حساب:

  * subtotal
  * discount
  * tax
  * total

#### 🔹 Payment System

* Cash / Card
* حساب الباقي

#### 🔹 Order Completion

* إرسال الطلب إلى API
* حفظ الفاتورة
* طباعة الفاتورة

---

## 5.2 📦 Product Management

### ⚙️ الوظائف:
